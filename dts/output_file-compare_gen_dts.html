Compare DTS and devicetree_generated.h

Following by last example, compare between DTS and devicetree_generated.h:

  1. root node "/": DT_N

    /*
 * Devicetree node: /
 *
 * Node identifier: DT_N
 */

/* Node's full path: */
#define DT_N_PATH "/"

/* Node's name with unit-address: */
#define DT_N_FULL_NAME "/"

...

  1. soc node "soc {": DT_N _S_ soc

    ...
/*
 * Devicetree node: /soc
 *
 * Node identifier: DT_N_S_soc
 */

/* Node's full path: */
#define DT_N_S_soc_PATH "/soc"

/* Node's name with unit-address: */
#define DT_N_S_soc_FULL_NAME "soc"

/* Node parent (/) identifier: */
#define DT_N_S_soc_PARENT DT_N
...
    • Node's full path: root node "/" + soc node "soc" = "/soc"
    • Node's name with unit-address: "soc {" = "soc"
    • Node parent (/) identifier: " root node identifier = DT_N

  1. gpio_000_036 node "gpio_000_036: gpio@40081000 {": DT_N _S_ soc _S_ gpio_40081000

    Warning

    the gpio node identifier is depending on node's name !

    Non-alphanumeric characters like dash (-) and the at sign (@) in devicetree names are converted to underscores (_). The names in a DTS are also converted to lowercase.

    reference : node identifier

    /*
 * Devicetree node: /soc/gpio@40081000
 *
 * Node identifier: DT_N_S_soc_S_gpio_40081000
 *
 * Binding (compatible = microchip,xec-gpio):
 *   $ZEPHYR_BASE/dts/bindings/gpio/microchip,xec-gpio.yaml
 *
 * (Descriptions have moved to the Devicetree Bindings Index
 * in the documentation.)
 */

/* Node's full path: */
#define DT_N_S_soc_S_gpio_40081000_PATH "/soc/gpio@40081000"

/* Node's name with unit-address: */
#define DT_N_S_soc_S_gpio_40081000_FULL_NAME "gpio@40081000"

/* Node parent (/soc) identifier: */
#define DT_N_S_soc_S_gpio_40081000_PARENT DT_N_S_soc
    • Node's full path: root node "/" + soc node "soc" + gpio node "gpio@40081000"= "/soc/gpio@40081000"
    • Node's name with unit-address: "gpio@40081000 {" = "gpio@40081000"
    • Node parent (/soc) identifier: " soc node identifier = DT_N _S_ soc

  1. compatible = "microchip,xec-gpio" property: DT_N_INST _0_ microchip_xec_gpio

    The name of the hardware device the node represents.

    Format: "vendor, device"

    ...
/* Existence and alternate IDs: */
#define DT_N_S_soc_S_gpio_40081000_EXISTS 1
#define DT_N_INST_0_microchip_xec_gpio DT_N_S_soc_S_gpio_40081000
#define DT_N_NODELABEL_gpio_000_036    DT_N_S_soc_S_gpio_40081000
...

    • DT_N_S_soc_S_gpio_40081000_EXISTS: only appear when defined / { soc { gpio_000_036: gpio@40081000 {

    • DT_N_INST _0_ microchip_xec_gpio:

      • INST: Get the number (ex. _0_, sorting by defined order) of instances of a given compatible(microchip_xec_gpio) with status okay
      • _0_: number of instances
      • microchip_xec_gpio: The name of the hardware device the node represents. Format is "vendor, device"

    • DT_N_NODELABEL_gpio_000_036: node label name from node's gpio_000_036: gpio@40081000 {

  1. reg= < 0x40081000 0x80 >:

    The reg property is a sequence of (address, length) pairs. Each pair is called a “register block”. Values are conventionally written in hex.

    ...
/* Macros for properties that are special in the specification: */
#define DT_N_S_soc_S_gpio_40081000_REG_NUM 1
#define DT_N_S_soc_S_gpio_40081000_REG_IDX_0_EXISTS 1
#define DT_N_S_soc_S_gpio_40081000_REG_IDX_0_VAL_ADDRESS 1074270208 /* 0x40081000 */
#define DT_N_S_soc_S_gpio_40081000_REG_IDX_0_VAL_SIZE 128 /* 0x80 */
...
    • DT_N_S_soc_S_gpio_40081000: gpio_4008100 node identifier
    • _REG: properties about information used to address the device.
    • _NUM: Get the number of register blocks in the reg property. Only 1 reg property specified as reg = < 0x40081000 0x80 >.
    • _IDX_0: the first index of reg property (max index: ${node_identifier}_REG_NUM - 1).
    • _VAL: value of "register block".
    • _EXISTS: only appear when defined the first reg property / { soc { gpio_000_036: gpio@40081000 { reg = < 0x40081000 0x80 >;
    • _ADDRESS: first parameter of reg property " reg = < ..._ADDRESS ..._SIZE > ". Displayed as decimal value.
    • _SIZE: second parameter of reg property. " reg = < ..._ADDRESS ..._SIZE > ". Displayed as decimal value.
