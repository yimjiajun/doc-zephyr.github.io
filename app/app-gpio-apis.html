GPIO APIs

To invokes GPIO APIs in ${ZEPHYR_BASE}/include/zephyr/drivers/gpio.h

Example: Set GPIO to High as output pin

in gpio.h, we should find the GPIO provided APIs:

1. Configuration GPIO pin:

  • Find GPIO configuration APIs to set pin as output:

    /**
* @brief Configure a single pin.
...
__syscall int gpio_pin_configure(const struct device *port,
       gpio_pin_t pin,
       gpio_flags_t flags);

static inline int z_impl_gpio_pin_configure(const struct device *port,
            gpio_pin_t pin,
            gpio_flags_t flags)
{

  • To setup GPIO as output pin, we should find flags for output pin:

    /**
 * @name GPIO input/output configuration flags
 * @{
 */
...
/** Enables pin as output, no change to the output state. */
#define GPIO_OUTPUT             (1U << 17)

  • Usage of GPIO configuration APIs in c source (ex. main.c), combine with gpio_pin_configure and GPIO_OUTPUT:

    const struct device *dev = DEVICE_DT_GET_OR_NULL(DT_ALIAS(one_of_gpio_port));

gpio_pin_configure(dev, SOME_OF_GPIO_PIN, GPIO_OUTPUT);

2. Set GPIO as level high:

  • Find GPIO APIs to set pin as level high:

    /**
 * @brief Set logical level of an output pin.
...
*/
static inline int gpio_pin_set(const struct device *port, gpio_pin_t pin,
             int value)
{

  • Usage of GPIO pin setup to level high:

    const struct device *dev = DEVICE_DT_GET_OR_NULL(DT_ALIAS(one_of_gpio_port));

gpio_pin_set(dev, SOME_OF_GPIO_PIN, 1);	//Level high
gpio_pin_set(dev, SOME_OF_GPIO_PIN, 0);	//Level low

3. Get Device information (Ex. MEC1501)

  • Device hardware component is provided by MEC1501 in path: ${ZEPHYR_BASE}/dts/arm/microchip/mec1501hsz.dtsi

    All hardware component descriptions will belong in folder dts/

    MEC1501 is belong to microchip family which is 32bit ARM architecture.

  • Retrieves one of GPIO port which multiples pin grouped by microchip for GPIO pin 0 to 36, the node label named as gpio_000_036 as show below:

    soc {
  ...
  gpio_000_036: gpio@40081000 {
    compatible = "microchip,xec-gpio";
    reg = < 0x40081000 0x80 0x40081300 0x04
            0x40081380 0x04 0x400813fc 0x04>;
    interrupts = <3 2>;
    gpio-controller;
    port-id = <0>;
    girq-id = <11>;
    #gpio-cells=<2>;
  };
  ...
}

  • Retrieves device information

    const struct device *dev = DEVICE_DT_GET_OR_NULL(DT_NODELABEL(gpio_000_036));

if (device_is_ready(dev) == 0)
  return -1;
    namedescription
    gpio_000_036device node label
    DT_NODELABEL()retrives node label as full node path name
    DEVICE_DT_GET_OR_NULL()retrieves const struct device* from a devicetree node of given full node path name if device available else retrives nothing
